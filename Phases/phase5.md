# 5️⃣ Decomposed Phase 5: Frontend Development and User Interface

This phase creates the user's interaction point with the system.

## 5.1.1 React/Tailwind Project Init
- **Purpose**: Scaffold the frontend application.
- **Detailed Steps**: Initialize a Vite project with React and TypeScript. Configure Tailwind CSS and a base theme (Dark/Light).
- **Tools/Technologies**: `Vite`, `React`, `Tailwind`
- **Expected Output**: Clean, empty project structure with a running dev server.

## 5.2.1 Resume Upload & Parsing UI
- **Purpose**: Handle the initial user intake.
- **Detailed Steps**: Build a file drop zone and a loading state that calls the ResumeParser API.
- **Tools/Technologies**: `React`, `Axios`
- **Expected Output**: Upload component that displays extracted skills upon completion.

## 5.2.2 Skill Radar Visualization
- **Purpose**: Provide immediate visual feedback of the user's profile.
- **Detailed Steps**: Use recharts to render a 5-point radar chart of the user's skill proficiencies.
- **Tools/Technologies**: `Recharts`
- **Expected Output**: Interactive Radar Chart component.

## 5.3.1 Target Selection & Constraints Form
- **Purpose**: Capture user goals and limits.
- **Detailed Steps**: Create a form with a searchable Job Role dropdown and sliders for Budget and Time Cap.
- **Tools/Technologies**: `React Hook Form`
- **Expected Output**: Goal-setting UI with validation.

## 5.3.2 Timeline Roadmap Component
- **Purpose**: Visualize the career path.
- **Detailed Steps**: Create a vertical timeline component that renders each course/project as a "milestone card."
- **Tools/Technologies**: `Tailwind CSS`, `Framer Motion`
- **Expected Output**: A sequential list of steps with cost/time labels.

## 5.3.3 Predicted Resume View
- **Purpose**: Show the "future self" value proposition.
- **Detailed Steps**: Create a secondary resume template that highlights the new skills the user will gain upon completion.
- **Tools/Technologies**: `React`
- **Expected Output**: A "Before vs. After" resume comparison view.
